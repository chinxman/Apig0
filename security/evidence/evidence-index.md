# Evidence Index

This index tracks Apig0 security-readiness evidence. Entries marked `Pending` must not be treated as completed validation.

## Evidence Register

| Evidence ID | Evidence Type | Location | Origin | Status | Notes |
| --- | --- | --- | --- | --- | --- |
| EV-001 | Local command history | `security/evidence/commands-run.md` | Apig0 repository | Pending | Populated by local scripts as checks are run. |
| EV-002 | Local scan outputs | `security/scans/` | Apig0 repository | Pending | Generated by `security/scripts/run-local-security-checks.sh`. |
| EV-003 | External security lab outputs | `security/evidence/imported/` | `apig0-security-lab` | Pending external lab initialization | Imported only through the approved evidence workflow. |
| EV-004 | Screenshots | `security/evidence/screenshots/` | Manual or external lab | Pending | Store only screenshots that support a finding or retest. |
| EV-005 | Runtime logs | `security/evidence/logs/` | Local Apig0 runtime | Pending | Store only logs relevant to security validation or incident review. |
| EV-006 | Evidence bundles | `security/evidence/bundles/` | Apig0 repository | Pending | Created by `security/scripts/collect-evidence.sh`. |

## Handling Rules

- Do not overwrite original evidence without preserving a backup.
- Record source tooling and scan origin for imported artifacts.
- Link findings to evidence IDs when validation is complete.
- Keep pending entries clearly marked until the supporting artifact exists.
